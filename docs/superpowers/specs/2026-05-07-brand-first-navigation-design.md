# 브랜드 우선 네비게이션 재구조 (Brand-First Navigation)

작성일: 2026-05-07
대상 파일: `src/App.js`

## 배경 / 목적

기존 최상위 메뉴는 `[원가 | 광고 | 결산]` 기능 축이었다. 실제 운영에서는 "팔레오 자사몰의 매출은?" 처럼 **브랜드와 쇼핑몰이 1차 컨텍스트**고, 매출/광고/원가는 그 안에서 보는 보조 축이다. 메뉴 위계를 실사용 흐름에 맞춰 재배치한다.

## 새 정보 위계

```
Brand (사이드바)
  └ Mall (본문 1단 탭: 자사몰 / 스마트스토어)
       └ Main (본문 2단 탭: 매출 / 광고 / 원가)
            └ (매출만) Sub (3단 탭: 주문입력 / 주문조회 / 결산조회)
```

## 사이드바

```
[메뉴]                    ← 접기 토글 유지
─── BRANDS ────────────
  🟢 팔레오              ← 클릭 시 currentBrandId 갱신, 강조 표시
  🟠 아프리모
  🔵 코코엘
─── 기타 ──────────────
  🔔 공지사항
  📋 상품소개서
  🎬 유튜브협찬
  🎯 팝업관리 (admin)
  🔒 보안 (admin)
```

- 기존 최상위 `[원가 | 광고 | 결산]` 버튼 제거
- 기존 BRANDS의 "브랜드 펼치기 → 쇼핑몰 클릭 → 결산 필터 적용" 인터랙션 제거 (단순 클릭으로 브랜드 선택만)
- 권한/부서 필터링은 기존 로직 그대로 (admin/director = 전 브랜드, 그 외 = 본인 부서 브랜드)

## 본문 영역

브랜드가 선택된 경우:

```
┌─────────────────────────────────────────────┐
│ 🟢 팔레오                                    │  헤더 (브랜드명 + 컬러)
├─────────────────────────────────────────────┤
│ [ 자사몰 ] [ 스마트스토어 ]                   │  Mall 탭
├─────────────────────────────────────────────┤
│ [ 매출 ] [ 광고 ] [ 원가 ]                    │  Main 탭
├─────────────────────────────────────────────┤
│ [주문입력] [주문조회] [결산조회]              │  매출 선택시만
├─────────────────────────────────────────────┤
│   본문 콘텐츠                                 │
└─────────────────────────────────────────────┘
```

- Mall 탭은 **항상 [자사몰, 스마트스토어] 2개 고정 노출**
- 미연동 쇼핑몰(예: 아프리모 스마트스토어) 클릭 가능, 본문에서 안내:
  > ℹ️ 아프리모 스마트스토어는 아직 연동되지 않았습니다. (준비 중)
- 미연동 판단: `brand.mallTypes` 배열에 `currentMallType` 포함 여부

## 모바일 하단 네비

```
[ 매출 ]  [ 광고 ]  [ 원가 ]
```

- 현재 선택된 브랜드/쇼핑몰의 main 탭 전환만 담당
- 브랜드 변경은 햄버거 메뉴(사이드바)에서, 쇼핑몰 변경은 본문 상단 Mall 탭에서 (데스크탑과 동일 위치)

## 상태 관리

### 제거되는 state
```js
const [tab, setTab] = useState("결산");
const [subTab, setSubTab] = useState("결산조회");
```

### 신규 state
```js
const [currentBrandId, setCurrentBrandId] = useState(null);
const [currentMallType, setCurrentMallType] = useState("자사몰");
const [mainTab, setMainTab] = useState("매출");          // 매출 / 광고 / 원가
const [salesSubTab, setSalesSubTab] = useState("결산조회"); // 주문입력 / 주문조회 / 결산조회
```

### 기존 `filter` / `pendingFilter`
- `brandId`/`mallType` 필드는 `currentBrandId`/`currentMallType`로 자동 동기화 (외부 탭이 단일 source-of-truth)
- 매출 하위 화면(특히 결산조회) 내부의 **브랜드/쇼핑몰 드롭다운 필터 제거**
- 날짜 필터는 본문에 그대로 유지

## 동작 흐름

1. **로그인 직후**: 사용자가 볼 수 있는 첫 번째 브랜드를 `currentBrandId`로 자동 선택. 기본값: `currentMallType="자사몰"`, `mainTab="매출"`, `salesSubTab="결산조회"`
2. **브랜드 미선택 상태**(예: 사용자에게 허용된 브랜드 0개): 본문에 "조회 권한이 있는 브랜드가 없습니다" 안내
3. **브랜드 변경**: 사이드바 클릭 → `currentBrandId`만 갱신. `currentMallType` / `mainTab` / `salesSubTab`는 유지 (다른 브랜드로 같은 화면 비교 편의)
4. **쇼핑몰 변경**: 본문 1단 탭 클릭 → `currentMallType` 갱신
5. **메인탭 변경**: 본문 2단 탭 클릭 → `mainTab` 갱신
6. **매출 서브탭**: 본문 3단 탭 클릭 → `salesSubTab` 갱신
