### 1. 브랜드 우선 네비게이션 재구조 (`ce40204`)

- 본문 위계: 브랜드 → 쇼핑몰(자사몰/스마트스토어) → 매출/광고/원가 탭
- 매출 탭 안에 [주문입력 / 주문조회 / 결산조회] 서브탭
- 결산 본문에서 브랜드/쇼핑몰 셀렉터 제거 (사이드바가 단일 source-of-truth)
- 모바일 하단 네비를 [매출/광고/원가]로 변경
- State 모델: `currentBrandId`, `currentMallType`, `mainTab`, `salesSubTab` 도입
- Spec/Plan: `docs/superpowers/specs/2026-05-07-brand-first-navigation-design.md`

### 2. 사이드바 Mall Drawer (`dbf5f8b`, `aec128a`)

- mall 클릭 → drawer 닫히고 본문 풀 너비로 콘텐츠 표시
- 첫 로그인 시 자동 오픈, 다른 브랜드 클릭 시 재오픈

- 모바일: bottom sheet (외부 클릭 시 닫힘)
- 🔗 동기화 버튼은 drawer 안으로 이동
- Spec/Plan: `docs/superpowers/specs/2026-05-07-sidebar-mall-drawer-design.md`

### 3. 팔레오 스마트스토어 2개 분리 (`08b1883`)
- 팔레오에 **브랜드스토어** + **도깨비나라** 두 SmartStore 운영 지원
- `mall_type`을 실제 스토어 이름으로 사용하는 데이터 모델 (자사몰만 카페24, 그 외 모두 Naver Commerce 변종)
- 백엔드: `STORE_CRED_ALIAS` 도입으로 (brandId, mallType) 튜플별 자격증명 분기
- proxy.js: 매핑없음 404, 자격증명누락 503, 인증실패 401로 에러 차별화
- sync.js: 자격증명 미설정 store는 경고+스킵으로 cron 안전성 확보
- Spec/Plan: `docs/superpowers/specs/2026-05-07-paleo-dual-smartstore-design.md`

### 4. 네이버 검색광고 1단계 연동 (`59249ec`, fix들 `783c897`/`4b82fdd`/`09d8023`)
- 팔레오 자사몰 네이버 검색광고 일별 광고비 + ROAS 표시
- `api/naver-ad.js` (Vercel 함수): HMAC-SHA256 시그니처로 Naver Search Ad API 호출
- `naver_ad_stats` 테이블 (일별 row, 1단계는 `campaign_id=''` sentinel)
- 광고 탭 라우팅 분기: 자사몰+등록 brand만 활성, 그 외 안내 메시지
- 본문: 요약 카드 4개(광고비/노출/클릭/CTR) + ROAS 카드 2종 + 일별 표
- 동기화 모달: 프리셋(최근7일/당월/전월) + 직접지정
- ROAS 라벨: **"자사몰 전체매출 ROAS"** (자사몰 매출 ÷ 광고비) + **"광고로 발생한 매출 ROAS"** (광고 전환매출 ÷ 광고비, Naver attributed)
- 시행착오:
  - `ids` 파라미터 JSON array → comma-separated 수정
  - `breakdown=day` 미지원 발견 → 일자별 loop API 호출로 전환
  - RLS 정책 누락 → SELECT/ALL 정책 추가
  - useEffect TDZ 버그 (final review 발견) → currentBrand 정의 이후로 위치 이동
- 운영 미점:
  - **자사몰 전체매출 ROAS는 Naver 광고비만 분모로 잡혀 부풀려진 수치** — Google/Meta/유튜브 등 다른 채널 광고비도 추가 후 의사결정 지표로 사용 가능 (다음 작업)
- Spec/Plan: `docs/superpowers/specs/2026-05-07-naver-ad-integration-design.md`

## 버그 수정

### 7. 코코엘 자사몰 동기화 0건 (`efc1d48`)
- **증상**: 12건 수집됐다고 나오지만 모두 "중복 12건 건너뜀"
- **원인**: PostgreSQL `21000` 에러 — `canceled=F`/`canceled=T` 양쪽 호출에서 같은 order_id가 `allOrders`에 중복 entry로 들어가서 한 batch upsert 내 같은 unique key가 두 번 update 대상이 됨
- **수정**: `allOrders`를 order_id 기준 Map dedup (canceled=T 후순위 entry 우선)

### 8. 강제 새로고침 시 브랜드 목록 비는 버그 (`faf9156`)
- **증상**: Ctrl+Shift+R 후 사이드바 비고 "조회 권한이 있는 브랜드가 없습니다" 표시. 로그아웃→재로그인하면 정상.
- **원인**: Supabase v2 lock contention — `onAuthStateChange` 콜백을 `async`로 선언하고 그 안에서 직접 `supabase.from(...)` 쿼리 시 supabase auth 내부 lock과 충돌해서 `profiles` 쿼리가 무한 대기. role 못 받아오니 admin도 manager로 fallback → visibleBrands 빈 배열.
- **수정**: 콜백을 sync로 만들고, 쿼리는 `setTimeout(..., 0)`으로 deferred 실행 (Supabase 팀 공식 권장 패턴)

### 네이버 검색광고 연동 (Vercel + Supabase)
**Vercel 환경변수**:
```
PALEO_NAVERAD_CUSTOMER_ID
PALEO_NAVERAD_ACCESS_LICENSE
PALEO_NAVERAD_SECRET_KEY
```
네이버 광고주센터에서 발급받은 자격증명. Production/Preview/Development 모두 적용.

**Supabase 신규 테이블**:
```sql
CREATE TABLE naver_ad_stats (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  brand_id uuid REFERENCES brands(id) ON DELETE CASCADE,
  mall_type text NOT NULL DEFAULT '자사몰',
  date date NOT NULL,
  campaign_id text NOT NULL DEFAULT '',  -- 1단계: '' sentinel, 2단계: 실제 캠페인 ID
  campaign_name text,
  impressions int DEFAULT 0,
  clicks int DEFAULT 0,
  cost bigint DEFAULT 0,
  conversions int DEFAULT 0,
  conversion_value bigint DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  UNIQUE(brand_id, date, campaign_id)
);
-- RLS 정책 (인증된 사용자 전체 read/write)
CREATE POLICY "naver_ad_stats_select" ON naver_ad_stats FOR SELECT TO authenticated USING (true);
CREATE POLICY "naver_ad_stats_all_write" ON naver_ad_stats FOR ALL TO authenticated USING (true) WITH CHECK (true);
```

**첫 동기화 검증**: 팔레오 자사몰 → 광고 탭 → 최근 7일 → 6일치 동기화 완료. 광고비 217,870원 / 클릭 251건 / 전환매출 4,328,200원 정상 수집.
