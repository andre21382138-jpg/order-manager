## 신규 기능 — 상품 탭

### 1. 상품 탭 신규 추가 (`0adb57b`)
- mainTab에 "🛒 상품" 추가 → 순서: 매출 / 광고 / **상품** / 원가
- 모바일 하단 네비도 동기 (4개 탭)
- 자사몰(카페24) + 카페24 토큰 등록 브랜드 한정 — 다른 mall은 "지원 안 됨" 안내
- 다른 브랜드(코코엘 등)도 카페24 연동되어 있으면 자동 동작
- 컬럼: 상품번호 / 상품명 / 판매가 / 정상가 / 공급가 / 상품구분 (초기값 "-")
- 검색 (상품명 + 상품번호 부분일치) + 모든 컬럼 헤더 클릭 정렬
- 빈 상태 / 로딩 / 에러 처리

### 2. 캐시 우선 로딩 (`3a05ac6` → `8f2773c`)
- 초기엔 탭 클릭마다 fetch 3~5초 발생 → 사용자 불편 피드백
- 개선 1차: state 캐시 + 명시적 "🔄 새로고침" 버튼
- 개선 2차: catalog_products 테이블 캐시 우선 로드 (DB만, 카페24 API 호출 없음)
  - 탭 진입 = catalog_products SELECT
  - 새로고침 버튼 = 카페24 API + catalog_products upsert + reload
  - **토큰 만료여도 캐시는 그대로 표시** (이전 데이터 보존)
  - "최근 갱신: M월 D일 HH:MM" 표시
- catalog_products 스키마 확장: `retail_price BIGINT DEFAULT 0` 컬럼 추가 (사용자 SQL 실행)

### 3. 토큰 자동 갱신 + OAuth 팝업 자동 연동 (`6911702` → `fc2bd45`)
- 1차 개선: refresh_token으로 access_token 자동 갱신 + Supabase 저장 + 로컬 state 갱신 → invalid_token 401 회피
- 2차 개선: **새로고침 버튼 한 번으로 재연동까지 자동**
  1. 1차 fetch 시도
  2. 토큰 만료 감지 (`invalid_token` / `unauthorized` / `401` / `invalid_grant`)
  3. 자동으로 카페24 OAuth 팝업 (600x700) 오픈 — 사이드바 안 가도 됨
  4. 사용자 로그인 → 팝업 닫힘 → 새 토큰 저장
  5. 자동 재시도 → catalog_products 저장 → 캐시 reload
- `openCafe24Auth(brand, mallId, opts)`가 `Promise<token>` 반환
- 팝업 강제 종료 감지 (interval polling, 500ms) → 무한 대기 방지

### 4. 상품구분 매핑 일괄 import (`abd8552` → `eafcbd5`)
- 헤더에 "📥 매핑 가져오기" 버튼 + 모달
- 1차: Excel 상품번호 ↔ Cafe24 product_no 매칭, 옵션 row 발산은 mode 채택
- **2차 변경**: Excel **주문상품명** ↔ Cafe24 **product_name** 정확 매칭
  - 이름 정규화: `<br>` 등 HTML 태그 제거 + 여분 공백 정리
  - 같은 카페24 상품에 여러 Excel row 매칭 시 mode
  - 옵션별 정확한 매핑 가능 (이전 product_no 기반보다 정확)
- 미리보기:
  - ✅ 이름 매칭 (저장 대상)
  - ⚠️ 카테고리 발산 — 펼쳐보기 (mode 선택 + 대안)
  - ❌ Excel-only (Cafe24 이름 매치 없음)
  - 📦 Cafe24-only — 펼쳐보기 (향후 인라인 편집 대상, 처음 100개)
- product_category_map에 50개씩 batch upsert
- 저장 후 상품 탭 자동 갱신 + 헤더에 "매핑 N/M" 카운트

### 5. 상품구분 정렬 (`da6fe8d`)
- 상품구분 헤더 클릭 → 가나다순 정렬 (한국어 `localeCompare`)
- 매핑 없는 "-" 상품은 asc/desc 무관 항상 **맨 끝**으로 → 매핑 안 된 신규 상품 찾기 쉬움
- 같은 카테고리끼리 묶여서 보임 (모아보기 효과)
- 텍스트 컬럼은 default 오름차순, 숫자 컬럼은 default 내림차순
