## 신규 기능

### 1. 네이버 검색광고 — 캠페인별 성과 (2단계, `1554a14` 등)
- 캠페인 단위 기간 합산 표 (`naver_ad_stats` dual-row 패턴: `campaign_id=''` 일별 + `campaign_id='cmp-...'` 캠페인별)
- 동기화 시 `/ncc/campaigns` + 일별 `/stats` per-day 루프로 캠페인 메타 + stats 수집
- 응답 구조 `{ stats:[], campaigns:[], _debug:{} }`로 변경
- 컬럼: 캠페인명 / 광고비 / 노출 / 클릭 / CTR / 전환수 / 전환매출 / ROAS
- 정렬: 광고비 내림차순 기본
- 광고비 0인 캠페인 자동 제외
- Spec/Plan: `docs/superpowers/specs/2026-05-08-naver-ad-campaign-stats-design.md` + `plans/2026-05-08-naver-ad-campaign-stats.md`

### 2. 광고영역(`campaign_type`) 추가 (`646e424`, `820fd72`)
- DB 컬럼 `naver_ad_stats.campaign_type` 추가 (TEXT NULL)
- API에서 캠페인 메타의 `campaignTp` 추출하여 row마다 저장
- `CAMPAIGN_TYPE_LABEL` 매핑: WEB_SITE→파워링크, SHOPPING→쇼핑검색, POWER_CONTENTS→파워컨텐츠, BRAND_SEARCH→브랜드검색, PLACE→플레이스, POWER_LINK_PRO→파워링크 PRO
- **광고영역별 광고 성과 카드** 신규 추가 — 광고영역별 합산 + ROAS
- 사용자 피드백으로 "타입" → "광고영역" 용어 통일

### 3. 캠페인 검색 + 엑셀형 필터 + 컬럼 정렬 (`1fc2c2a`, `790028c`, `46fe9ac`)
- 캠페인명 부분일치 검색 input
- 광고영역 컬럼 헤더 ▼ → 체크박스 팝업, 다중 선택 가능, 활성 시 ▼ 파란색 + "✕ 광고영역 필터 해제" 단축 버튼
- 모든 숫자 컬럼(광고비/노출/클릭/CTR/CPC/전환수/전환매출/ROAS) 헤더 클릭 정렬 (▼ 내림 / ▲ 오름 토글)
- 활성 정렬 컬럼은 파란색

### 4. CPC 컬럼 + 표 빈 상태 처리 개선 (`2b43f6c`, `19ac769`)
- 모든 광고 표(일별/광고영역별/캠페인별)에 CPC 컬럼 추가
- 광고영역 필터로 표가 비어도 헤더는 유지(▼/필터 해제 버튼 항상 접근 가능)
- 빈 상태는 `tbody`의 `colSpan` row로 표시

### 5. 네이버 검색광고 — 키워드별 성과 (3단계 Phase 1, `8d26bed`~`6b3d6bf`)
- **A. 낭비 키워드 식별** + **B. 성공 키워드 푸시** 사용 케이스
- 신규 테이블 `naver_ad_keyword_stats` (truncate-and-insert per brand+mall, UNIQUE(brand_id, mall_type, keyword_id))
- API `action=keywords`: 활성 캠페인 → 활성 광고그룹 → 활성 키워드 사전 필터로 호출 수 80% 감소
- bulk `/stats` (100개 chunk, parallel 5)로 키워드 통계 압축
- Frontend: 동기화 모달에 "🔑 키워드까지 동기화" 체크박스 (default off)
- 11컬럼 표: 키워드 / 캠페인 / 광고영역 / 광고비 / 노출 / 클릭 / CTR / CPC / 전환수 / 전환매출 / ROAS
- 검색 + 캠페인 필터(체크박스 팝업) + 모든 숫자 컬럼 정렬 (캠페인별 표와 동일 UX)
- 운영 검증: 팔레오 7일 기간 → 캠페인 67개 → 광고그룹 200+ → 키워드 124개 / 21초
- Spec/Plan: `docs/superpowers/specs/2026-05-08-naver-ad-keyword-stats-design.md` + `plans/2026-05-08-naver-ad-keyword-stats.md`

## 버그 수정

### 8. 키워드 stats — 광고그룹 호출 chunk 누락 (`0ab9463`)
- **증상**: 키워드 동기화 중 "❌ 키워드 동기화 실패: adgroup stats fetch 실패"
- **원인**: 활성 캠페인 67개 → 광고그룹 200+개를 단일 `/stats`에 모아 호출 → Naver API ID 한도/URL 길이 초과
- **수정**: Step 4(광고그룹 stats)도 100개 chunk + parallel 5 (Step 6 키워드와 동일 패턴). Step 2(캠페인 stats)도 방어적으로 동일 처리.
- 프론트엔드 에러 메시지에 `data.raw` / `data._debug` 일부 노출하여 다음 디버깅 즉시 가능

### 9. 키워드 응답 부분실패 관찰성 (`33d4053`)
- 광고그룹/키워드 fetch는 일부 실패해도 빈 배열 반환 → silent 손실
- `_debug.warnings` 배열에 `{stage, id, status}` 기록
- 광고그룹 stats 단일 호출 실패는 silent 빈 결과 대신 명시적 5xx 응답으로 변경

### 10. 키워드 useEffect dead dependency (`6b3d6bf`)
- `mall_type`을 "자사몰" 하드코딩하면서 deps에 `currentMallType` 포함 → 무의미한 refetch
- deps에서 제거

### 11. sync 후 filter 범위 stale 우회 (`939b247`)
- 동기화 모달의 기간이 현재 filter 범위 밖이면 동기화 직후 표가 비어 보이는 버그
- 동기화 성공 시 `filter.from/to`를 sync 범위까지 확장 (기존 범위가 더 넓으면 유지)

### 15. 자동 동기화 cron (`7205aa1`, `dc96eaa`)

## 워크플로

오늘 작업 패턴 정리:

- 큰 기능(캠페인별, 키워드별, 일자별, 트렌드, cron)은 **brainstorming → spec → plan → subagent-driven-development** 풀 사이클로 진행
- 작은 UI 개선(검색·필터·정렬·CPC·ROAS 카드 제거·필터 draft 패턴·모달 흐름 분리)은 사용자 피드백 즉시 → 직접 구현 → 빌드 → 커밋
- 코드 품질 리뷰 피드백은 controller가 직접 fix (SendMessage 미지원)
- 운영 검증에서 발견된 버그는 systematic-debugging 4-phase로 즉시 fix
- 사용자 통찰("sync 한 번 → 다양한 뷰")은 memory에 저장하여 향후 설계 결정에 반영
